Issue #130 (chat assistant bubbles show tool-call XML as raw text, bare URLs aren't clickable) was a display-only bug. The model emits tool invocations inline in the text stream — `<kody_run_issue />` for self-closing and `<tool_call>…</tool_call>` blocks for JSON payloads. The structured call is already captured separately and rendered via `ThinkingPanel`, so the raw markup leaking into the visible answer is just noise the user has to mentally filter out. Bare URLs were plain text because `ReactMarkdown` was rendered without `remark-gfm`.

The fix is in two places, both in the render path of the assistant bubble in `src/dashboard/lib/components/KodyChat.tsx`. A new helper at `src/dashboard/lib/chat/tool-call-strip.ts` exports `stripToolCallMarkup` (known-tool-name tag stripper) and `parseAssistantContent` (wraps `parseReasoning` and applies the stripper to the answer). The bubble now destructures from `parseAssistantContent` instead of `parseReasoning`, and `ReactMarkdown` now has `remarkPlugins={[remarkGfm]}` plus a custom `a` component that opens external links in a new tab with `rel="noopener noreferrer"` (matches `CommentList` and `MessagesView`).

Tests: new `tests/unit/chat/tool-call-strip.spec.ts` covers the stripper (closed/unclosed `<tool_call>` blocks, self-closing tags with and without attrs, partial streaming tails, plain-text preservation, blank-line collapse, combined reasoning+tool-call stripping); new `tests/unit/chat/kody-chat-bubble-tool-call-markup.spec.ts` is a source-level structural test that pins the import + `remarkPlugins` wiring so a future refactor can't silently drop the fix. `pnpm verify` (typecheck + lint + format + tests) passed.
